# ADR-0001: Подход к стилям — CSS Modules

**Статус:** принято
**Дата:** 2026-07-02

## Контекст

Спека (`sudoku-pwa-spec.md` §2) оставляет выбор системы стилей за исполнителем: «CSS Modules или Tailwind (на выбор агента, зафиксировать в начале)». Решение сквозное — влияет на все компоненты, сборку и стиль кода.

Действующие силы:
- CLAUDE.md ставит читаемость высшим приоритетом и требует «flat and scannable JSX» — плотные utility-классы в разметке этому противоречат.
- Ключевой визуальный элемент — адаптивная сетка 9×9 (CSS Grid, `aspect-ratio`), которую удобнее описывать отдельным CSS.
- Vite поддерживает CSS Modules из коробки, без дополнительной сборочной цепочки.

## Решение

Используем **CSS Modules** (`*.module.css`) рядом с компонентами. Tailwind не подключаем.

## Последствия

- Проще: JSX остаётся плоским и читаемым; нет PostCSS/конфига Tailwind; нативная поддержка Vite.
- Сложнее: нет готовых utility-классов — общие токены (цвета, отступы) держим в CSS-переменных, чтобы избежать дублирования.
- Закрывает развилку §2 спеки; при желании ввести тему оформления (backlog) CSS-переменные уже дадут точку расширения.
