# ADR-0003: Архитектура состояния и слоёв

**Статус:** принято
**Дата:** 2026-07-02

## Контекст

Нужно определить сквозную структуру состояния и границы слоёв. Спека (§2) фиксирует React Context + useReducer (Redux избыточен) и (§7) выделяет чистый слой `core/`. В ходе брейншторминга добавился второй экран (статистика), а спека при этом требует «без роутинга».

Действующие силы:
- Критичная логика судоку должна тестироваться изолированно, без React и без DOM.
- Объём состояния средний — Redux избыточен.
- Экранов всего два (игра / статистика) — полноценный роутер избыточен.
- CLAUDE.md требует замены switch-цепочек на lookup-структуры.

## Решение

Три слоя с однонаправленными зависимостями `components → state → core`:
- **`core/`** — чистые функции (solver, generator, validator), без React и хранилищ.
- **`state/`** — `gameReducer` (диспетчеризация actions через `Record<ActionType, handler>`, не switch), сервисы статистики и обёртки хранилищ, провайдеры Context.
- **`components/`** — тонкий UI без игровой логики.

Навигация между экранами — поле `activeView: 'game' | 'stats'` в Context, **без react-router**.

## Последствия

- Проще: `core/` покрывается unit-тестами напрямую; редьюсер тестируется без DOM; нет зависимости от роутера.
- Сложнее: при появлении третьего+ экрана или потребности в deep-linking переключатель видов придётся заменить на роутер (пересмотр этого ADR).
- Соответствует приоритету читаемости: логика вынесена из компонентов, диспетчеризация декларативна.
