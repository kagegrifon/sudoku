# ADR-0004: Стратегия обновления PWA

**Статус:** принято
**Дата:** 2026-07-02

## Контекст

`vite-plugin-pwa` предлагает два режима обновления service worker: `autoUpdate` (новый SW активируется и перезагружает страницу автоматически) и `prompt` (новый SW ждёт в состоянии `waiting`, активация — по команде приложения). Решение сквозное: влияет на конфиг SW, состояние приложения и UX.

Действующие силы:
- Игра хранит незавершённую партию; автоматическая перезагрузка посреди партии — плохой UX, а на iOS поведение автоперезагрузки непредсказуемо.
- Партия переживает обновление (localStorage + восстановление), но при debounce-сохранении последний ход может не успеть записаться до reload.
- Схемы хранилищ версионируются (см. [ADR-0002](0002-storage-strategy.md)).

## Решение

Используем **`registerType: 'prompt'`**. При готовности новой версии (`onNeedRefresh`):
1. показываем всплывающий баннер «Доступно обновление»;
2. если игрок его закрыл — сворачиваем в постоянную кнопку «Обновить» в хедере (флаг `updateReady` сохраняется);
3. по кнопке форсируем flush сохранения партии в localStorage, затем `updateServiceWorker(true)` (skipWaiting + reload).

## Дополнение (2026-07-19): ручная проверка

Автоматическое обнаружение оставляет пробел: пока браузер не проверил `sw.js`,
игрок видит задизейбленную кнопку и не может ничего сделать. В standalone-режиме,
который редко закрывают полностью, проверка откладывается надолго.

Добавлена ручная проверка — та же кнопка в строке «Версия» меняет роль по состоянию
(`idle` → «Проверить обновления», `checking` → «Проверяем…», `updateReady` → «Обновить»)
и зовёт `registration.update()`. Три различимых исхода: «Нет соединения» (офлайн),
«Не удалось проверить» (запрос упал), «Обновлений не найдено» (10 с без `needRefresh`).

Формулировка «Обновлений не найдено», а не «Актуальная версия»: `update()` не сообщает
результат, поэтому по таймауту нельзя утверждать, что новой версии нет — она могла не
успеть скачаться. Спека: [manual-update-check](../superpowers/specs/2026-07-19-manual-update-check-design.md).

## Последствия

- Проще для игрока: обновление не рвёт партию, момент выбирает пользователь.
- Надёжнее на iOS: контролируемая активация вместо капризной автоперезагрузки.
- Сложнее: нужен модуль-обёртка над `virtual:pwa-register`, флаг `updateReady` в состоянии и UI (баннер + кнопка).
